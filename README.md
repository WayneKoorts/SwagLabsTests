# Swag Labs Tests

[![Tests](https://github.com/WayneKoorts/SwagLabsTests/actions/workflows/tests.yml/badge.svg)](https://github.com/WayneKoorts/SwagLabsTests/actions/workflows/tests.yml)

Playwright-based UI tests for [Swag Labs](https://www.saucedemo.com/) using NUnit and .NET.

## Prerequisites

- [.NET 10 SDK](https://dotnet.microsoft.com/download)
- [PowerShell](https://github.com/PowerShell/PowerShell) (for installing Playwright browsers)

## Setup

1. Build the project (this also restores NuGet packages):

   ```bash
   dotnet build src/SwagLabsTests.sln
   ```

2. Install Playwright browsers (required once, and after Playwright version updates):

   ```bash
   cd src/SwagLabsTests
   pwsh bin/Debug/net10.0/playwright.ps1 install
   cd ../..
   ```

   This is the [standard Playwright .NET setup step](https://playwright.dev/dotnet/docs/intro). The script is generated by the build and installs the browser binaries that Playwright needs.

3. Copy the environment file and adjust as needed:

   ```bash
   cp .env.example .env
   ```

   See `.env.example` for available options. The VS Code launch configuration (`.vscode/launch.json`) loads this file automatically via `envFile`.

## Running Tests

Run all tests:

```bash
dotnet test src/SwagLabsTests.sln
```

Run with detailed output:

```bash
dotnet test src/SwagLabsTests.sln --logger "console;verbosity=detailed"
```

Run a specific test:

```bash
dotnet test src/SwagLabsTests.sln --filter "HomePageShowsLoginForm"
```

### Headed Mode

To see the browser while tests run, uncomment `HEADED=1` in your `.env` file, or pass it inline:

```bash
HEADED=1 dotnet test src/SwagLabsTests.sln
```

### Slow Motion

To slow down each Playwright action (useful for watching tests in headed mode), pass `SlowMo` via the `--` runsettings override:

```bash
HEADED=1 dotnet test src/SwagLabsTests.sln -- Playwright.LaunchOptions.SlowMo=1500
```

The value is in milliseconds.
